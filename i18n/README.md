# vpsf-status translations

English source strings live in `internal/i18n/catalog/messages.go`.

Editable translations live in `<lang>.toml`, for example `cs.toml`. The
`*.active.toml` files are generated by `make i18n-update`, embedded into the Go
binary, and checked by `make i18n-health`.

## Czech guidelines

vpsf-status is read by server administrators, developers, and other IT staff.
Translate with infrastructure context in mind and keep common technical terms
in English when Czech would be unnatural.

- `Status` is `Status`.
- `State` is `Stav`.
- Disk or resource `free` is `volné` or `volno`, never `zdarma`.
- `Node`/`Nodes` are `Node`/`Nody`, not `uzel`.
- `Cluster` is `Cluster`, not `klastr`.
- `Kernel` is `Kernel`.
- `Monitoring` is `Monitoring`.
- `Storage` is `Úložiště`.
- `Network`/`Networks` are `Síť`/`Sítě`.
- `DNS resolvers` are `DNS resolvery`.
- `Name servers` are `DNS servery`.
- A planned outage is `odstávka`; an unplanned outage is `výpadek`.
  Never use `plánovaný výpadek`.
- `Back` navigation links are `Zpět`.
- `Event log` is usually `Události`.
- Language names in the selector are native names, e.g. `English` and
  `Česky`, regardless of the current page language.

Keep established technical terms untranslated when natural, for example API,
DNS, VPS, NAS, IPv4, IPv6, ARC, scrub, resilver, snapshot, dataset, resolver,
token, login, and hostname.
